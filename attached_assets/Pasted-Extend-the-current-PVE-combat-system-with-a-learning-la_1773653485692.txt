Extend the current PVE combat system with a learning layer that improves enemy action prediction over time using stored battle logs and observed fight state.

Goal:
The current system already has:
- live telemetry capture
- enemy state tracking
- static ability catalogs
- Monte Carlo look-ahead simulation
- consumable strategy
- execution modes

Now add a learning layer that uses historical fight data to improve:
1. enemy action probabilities
2. target selection probabilities
3. consumable timing heuristics
4. confidence estimation
5. simulation realism

==================================================
HIGH-LEVEL REQUIREMENT
==================================================

The system must learn from stored fight logs without replacing the deterministic combat engine.

Important:
- do NOT replace the formula engine
- do NOT replace hardcoded ability catalog logic
- do NOT invent mechanics
- use learning only to improve probability estimates, priors, and confidence

The final architecture should be:

1. live telemetry layer
2. deterministic state engine
3. deterministic formula engine
4. historical learning layer
5. enemy policy estimator
6. Monte Carlo simulation
7. recommender
8. execution layer

==================================================
WHAT THE LEARNING LAYER SHOULD LEARN
==================================================

Learn from previous encounters:

A. Enemy action choice tendencies
Examples:
- Mama Boar chooses Rampage more often when 3 player units are alive
- Baby Boar chooses Nuzzle more often when Mama Boar is below a certain HP threshold
- Bad Motherclucker chooses Lay Egg more often when dead Baby Rocbocs exist
- Baby Rocboc chooses Cheep more often when Bad Motherclucker is threatened

B. Enemy target selection tendencies
Examples:
- prefer lowest HP target
- prefer frontline
- prefer channeling targets
- prefer taunted targets
- prefer random when no strong context exists

C. Ability reuse timing
Examples:
- average turns until ability reuse after Amnesia expires
- whether enemy tends to use an available ability immediately or delay it

D. Consumable timing patterns for player-side recommendation tuning
Examples:
- Health Vial improves survival most when used before predicted AoE turns
- Mana Vial is strongest when it unlocks high-value skill windows

E. Confidence estimation
Examples:
- predictions should be more confident when similar states have many historical matches
- predictions should be lower confidence when unresolved abilities or rare states occur

==================================================
LEARNING METHOD
==================================================

Start simple and robust.

Phase 1 learning implementation:
- frequency tables
- conditional probability tables
- state bucket matching
- smoothed priors

Do NOT jump directly to neural networks.

Example:
For Mama Boar, estimate:
P(action = Rampage | mana >= 4, 3 enemies alive, Wild Charge locked, not channeling)

For Baby Rocboc, estimate:
P(action = Cheep | Rocboc hp low, channeling enemy present)

Use Laplace/additive smoothing so sparse data does not produce zero-probability behavior.

==================================================
HOW TO INTEGRATE WITH THE EXISTING PREDICTOR
==================================================

Do NOT let the learning layer override hard legality rules.

Use this order:

1. determine legal actions deterministically
2. determine legal targets deterministically
3. use learned policy to assign probabilities only among legal actions
4. combine learned probabilities with existing heuristic priors
5. feed resulting distributions into Monte Carlo simulation

Suggested weighted combination:
finalProbability =
0.60 * learnedPolicyProbability +
0.40 * heuristicPriorProbability

If sample count is low, downweight learned policy and rely more on heuristic priors.

==================================================
CONFIDENCE HANDLING
==================================================

Add prediction confidence logic.

Confidence should increase when:
- many similar historical examples exist
- state features are well-covered
- no unresolved abilities affect the decision

Confidence should decrease when:
- sparse sample count
- unresolved ability present
- missing UI state
- ambiguous target geometry
- recent balance/version mismatch

Example:
{
  "predictionConfidence": 0.74,
  "confidenceReasons": [
    "86 similar historical states found",
    "all ability availability verified",
    "no unresolved enemy actions in legal set"
  ]
}

==================================================
MONTE CARLO INTEGRATION
==================================================

Update the Monte Carlo simulation so enemy actions are sampled from:

+ learned action probabilities
+ learned target probabilities

- broaden distributions
- rely more on heuristics

If confidence is high:
- follow learned distributions more closely
