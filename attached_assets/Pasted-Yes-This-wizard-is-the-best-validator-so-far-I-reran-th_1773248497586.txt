Second, **P.DEF and M.DEF are also right** in the app for this wizard: 50.8 and 142.0 match exactly.

Third, the wizard proves your **P.RED / M.RED bug**. The card shows:

* `P.RED = 10.16% = 50.8 / (50 * 10)`
* `M.RED = 28.40% = 142.0 / (50 * 10)`

Your app shows exactly half of those: 5.08% and 14.20%. So the app is almost certainly dividing by `100 * avgLevel` instead of `50 * avgLevel`.

Use:

```ts
pRedPct = 100 * pDef / (50 * avgBattleLevel)
mRedPct = 100 * mDef / (50 * avgBattleLevel)
```

or equivalently:

```ts
pRedPct = 2 * pDef / avgBattleLevel
mRedPct = 2 * mDef / avgBattleLevel
```

This same relationship also fit the Priest card you sent earlier, so this is now a strong fix, not a guess.

The next strong finding is **EVA**. For the wizard:

For the priest:

* workbook-style base EVA was about **3.0**
* card EVA was **8.22**

That recurring +5.2 across both heroes is too clean to ignore. The official armor schema has a direct `evasion` field on armor (`DisplayDefenseInfo.evasion`), separate from bonus-code mappings, so this looks like your app is probably **not adding armor EVA** yet. ([DeFi Kingdoms Developers][1])

So the static formula should be:

```ts
evaStatic =
  evaDynamic(agi, lck, avgBattleLevel)
  + armor.displayDefenseInfo.evasion
  + armorBonus8
  + accessoryBonus6
  + passiveTraitEvasion
```

and then only after that do you apply live combat buffs/debuffs.

This also means two important fields are **not** bonus-code lookups:

* armor EVA comes from the armor’s explicit `evasion` field, not a bonus code, and
* weapon speed comes from the weapon’s explicit `speedModifier`, not a bonus code. The weapon docs define `speedModifier` and its conversion logic directly. ([DeFi Kingdoms Developers][2])

So your current architecture should separate:

```ts
displayedCombatStat =
  baseWorkbookFormula
  + staticEquipmentFields
  + staticEquipmentBonusCodes
  + passiveTraitBonuses
  + class/passive permanent effects
  + temporary battle FX
```
